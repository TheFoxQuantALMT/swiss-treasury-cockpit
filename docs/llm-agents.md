# LLM Agents (Daily Brief)

## Overview

The daily brief is generated by a two-agent pipeline using local Ollama models. This is optional -- the cockpit renders without it.

```
macro_snapshot.json + scores.json
         |
         v
    _build_template()     <-- Python pre-fills all numbers
         |
         v
    Analyst Agent          <-- LLM writes [INTERPRET] bullets only
    (deepseek-r1:14b)
         |
         v
    Reviewer Agent         <-- Programmatic fact-check + LLM review
    (qwen3.5:9b)
         |
         v
    generate_html_brief()  <-- Convert to styled HTML
         |
         v
    {date}_brief.json
```

## Design: Template-Fill Approach

The key design principle is **anti-hallucination**: Python pre-fills all numerical data into the template. The LLM only writes interpretive bullets (`[INTERPRET]` placeholders). This prevents the LLM from inventing numbers.

### Template Building

```python
from cockpit.agents.analyst import _build_template

template = _build_template(
    macro_data,     # raw fetched data
    deltas,         # 1d/1w/1m changes
    delta_table,    # formatted markdown table
    alerts,         # triggered alerts
)
```

The template contains:
- All numerical values pre-computed
- Delta tables with exact changes
- Alert descriptions
- `[INTERPRET]` markers where LLM commentary is needed

### Analyst Agent

```python
from cockpit.agents.analyst import create_analyst_agent

analyst = create_analyst_agent()
brief_text = asyncio.run(analyst.run(template))
```

- **Model:** `deepseek-r1:14b` (via Ollama)
- **Task:** Fill `[INTERPRET]` placeholders with market commentary
- **Constraint:** Must not change any pre-filled numbers

### Reviewer Agent

```python
from cockpit.agents.reviewer import programmatic_check, create_reviewer_agent

# Step 1: Programmatic fact-check
errors = programmatic_check(brief_text, macro_data)

# Step 2: LLM review (if programmatic check passes)
reviewer = create_reviewer_agent()
```

- **Model:** `qwen3.5:9b` (via Ollama)
- **Programmatic check:** Verifies all numbers in the brief match the source data
- **LLM review:** Checks coherence, completeness, tone
- **Retry loop:** Up to `MAX_REVIEW_RETRIES` (default: 3) attempts

### Reporter

```python
from cockpit.agents.reporter import generate_html_brief

html = generate_html_brief(brief_text, macro_data, deltas)
```

Converts the reviewed brief text into styled HTML for the dashboard brief tab.

## Requirements

```bash
# Install agent dependencies
uv sync --extra agents

# Start Ollama with required models
ollama pull deepseek-r1:14b
ollama pull qwen3.5:9b
ollama serve  # runs at http://localhost:11434
```

## Configuration

| Constant | Value | Description |
|----------|-------|-------------|
| `ANALYST_MODEL` | `deepseek-r1:14b` | Analyst LLM model |
| `REVIEWER_MODEL` | `qwen3.5:9b` | Reviewer LLM model |
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama server URL |
| `MAX_REVIEW_RETRIES` | 3 | Maximum review attempts |

## Graceful Degradation

If Ollama is unavailable or agents fail:
- `cmd_analyze` exits with an error message
- `cmd_run_all` catches the error and continues to `render`
- The dashboard renders without the brief tab (placeholder shown)
